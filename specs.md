# Спецификация модулей приложения TaskManager

## Общая структура
Приложение разделено на три модуля:  
- `data.py` — отвечает за хранение и загрузку данных из файла `tasks.json`  
- `logic.py` — реализует бизнес-логику: добавление, изменение, удаление задач  
- `ui.py` — управляет взаимодействием с пользователем через консоль  

Все модули взаимодействуют между собой через вызовы функций, без глобальных переменных.  
Главная точка входа — `main.py`, который связывает их между собой.

---

## Модуль: data.py — Хранение и загрузка задач

**Назначение:** обеспечить надёжную работу с хранилищем данных в формате JSON.

**Описание работы:**
- При запуске приложения пытается прочитать файл `tasks.json`.
- Если файл не найден, возвращает пустой список.
- После любых изменений в списке задач данные сохраняются в тот же файл.

| Функция | Вход | Выход | Описание |
|----------|------|--------|-----------|
| `load_tasks()` | — | `List[Dict]` | Читает файл `tasks.json`. Если файла нет — возвращает пустой список. |
| `save_tasks(tasks)` | `List[Dict]` | `None` | Перезаписывает файл `tasks.json`, форматирует отступами `indent=4`. |

**Ошибки:**  
Если JSON повреждён — программа не падает, просто возвращает `[]`.  

---

## Модуль: logic.py — Бизнес-логика

**Назначение:** реализует основные операции с задачами — добавление, изменение статуса, удаление.

**Описание работы:**
- Каждой задаче присваивается уникальный `id` (инкремент).
- Статус по умолчанию `"новая"`.
- Поддерживаются статусы: `"новая"`, `"в работе"`, `"завершена"`.

| Функция | Вход | Выход | Описание |
|----------|------|--------|-----------|
| `add_task(tasks, task_data)` | `List[Dict]`, `Dict` | `None` | Добавляет новую задачу с ID и статусом `"новая"`. |
| `change_status(tasks, task_id, new_status)` | `List[Dict]`, `int`, `str` | `bool` | Меняет статус по ID. Возвращает `True`, если успешно. |
| `delete_task(tasks, task_id)` | `List[Dict]`, `int` | `List[Dict]` | Удаляет задачу по ID, возвращает обновлённый список. |

    "id": 3,

    "desc": "До 15 ноября",
    "status": "в работе"
