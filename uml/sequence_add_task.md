# Диаграмма последовательности — Добавление задачи

**Цель:** показать процесс добавления новой задачи пользователем в консольном приложении.

## Участники
- **User** — человек, использующий приложение  
- **UI** — модуль `ui.py`, отвечает за взаимодействие с пользователем  
- **Logic** — модуль `logic.py`, реализует бизнес-логику  
- **Data** — модуль `data.py`, сохраняет данные в JSON  

## Последовательность

1. **User** выбирает пункт меню `1. Добавить задачу`  
2. **UI** вызывает `get_task_input()` и запрашивает название и описание  
3. **UI** передаёт введённые данные в **Logic.add_task(tasks, task_data)**  
4. **Logic** создаёт задачу с новым `id` и статусом `"новая"`  
5. **Logic** добавляет её в список `tasks`  
6. **Logic** вызывает **Data.save_tasks(tasks)**  
7. **Data** записывает список задач в файл `tasks.json`  
8. **UI** выводит сообщение «Задача добавлена»
